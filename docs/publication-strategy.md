# Publication strategy

## Goal

Make the repository citable, reviewable, and academically legible from the first public release.

## Release infrastructure

Before the first public release:

- add a real `LICENSE`
- finalize `CITATION.cff`
- finalize `codemeta.json`
- connect Zenodo to GitHub
- publish clean release notes
- archive benchmark definitions

## Scholarly outputs

### 1. Architecture / benchmark preprint

Use this to explain:

- why standards-first matters
- what the benchmark protocol is
- how the reproducibility bundle works
- why the project is a reference implementation rather than a prompt collection

### 2. JOSS paper

Prepare the repo so it satisfies software-paper expectations:

- open source
- obvious research application
- clear docs
- issue tracker
- tests
- release history
- examples

### 3. Demonstration papers

Later papers can use the platform for domain-specific demonstrations:

- behavioral methods
- EEG/MEG reproducibility
- modeling workflows
- meta-analytic workflows

## Recommended visibility moves

- GitHub Pages docs
- Zenodo DOI per release
- workshop materials
- design-partner case studies
- conference tutorials
